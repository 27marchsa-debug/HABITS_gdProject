## Ground Rule
## 1. 회의 
| 항목       | 내용                                       |
| -------- | ---------------------------------------- |
| 정기 회의    | 매주 수요일 17:00~, 목요일 15:30~ 진행             |
| 사전 준비    | 회의 전 각자 진행 사항을 문서로 정리하고 충분히 숙지           |
| 스프린트 플래닝 | 매주 수요일 진행: 역할 분담, 마일스톤 검토, 추가 작업 정리 포함   |
| 스프린트 회고  | 플래닝 직전 10~15분 진행, 핵심 내용은 Ideation.md에 기록 |
| 회의 원칙    | 개선 중심, 명확·간결한 의사소통, 시간 준수                |

## 2. 절차 
| 항목       | 내용                                     |
| -------- | -------------------------------------- |
| 개발 방식    | 스크럼·애자일 방식을 기본으로 함                     |
| 단계 진행 기준 | 실현 가능성(feasibility)을 최우선으로 판단          |
| 이슈 대응    | 오류 또는 긴급 이슈 발생 시 즉시 팀 채널(카톡/노션/슬랙)에 공유 |

## 3. 문서화
| 항목    | 내용                                               |
| ----- | ------------------------------------------------ |
| 로드맵   | 가능한 상세하게 작성하여 방향성을 명확히 함                         |
| 회의 기록 | 매주 스크럼 회의 내용을 Notion에 기록 후 핵심만 Ideation.md에 업데이트 |
| 기술 문서 | 데이터 구조, API 명세, 아키텍처 등은 변경 시 즉시 업데이트             |
| 작성 원칙 | 명확성, 일관성, 최신성 유지                                 |

## 4. 백로그 
| 항목      | 내용                                        |
| ------- | ----------------------------------------- |
| 백로그 관리  | Notion 페이지에서 통합 관리                        |
| 변경 발생 시 | 작업/요구사항 변경 시 백로그 업데이트 → 우선순위 조정 → 마일스톤 반영 |
| 백로그 항목  | 작업 제목, 상세 내용, 담당자, 예상 난이도, 예상 완료일         |
| 완료 처리   | 완료된 작업은 회의에서 확인 후 Done 처리                 |


## 5. 협업 규칙 
| 항목    | 내용                               |
| ----- | -------------------------------- |
| 기록 원칙 | 중요한 논의는 반드시 기록으로 남김              |
| 응답 원칙 | 팀원 간 응답은 가능하면 24시간 이내 처리         |
| 코드 품질 | 코드 컨벤션 준수, 테스트 가능·재사용성 고려        |
| 업무 태도 | 맡은 업무는 책임감 있게 수행하며 문제 발생 시 즉시 공유 |


## 6. 리스크 관리
| 항목     | 내용                                   |
| ------ | ------------------------------------ |
| 리스크 공유 | 일정 지연, 기술적 난관 등은 주간 회의에서 공유          |
| 일정 조정  | 예기치 않은 문제 발생 시 일정/계획을 유연하게 조정        |
| 의사 결정  | 결정이 필요한 사안은 다수 의견 또는 팀 리더 판단으로 신속 결정 |
